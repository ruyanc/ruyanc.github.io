外部驱动数据类型汇总

冷启动运行 RAP（就是没有已有 RAP 周期循环状态或者有限／缺失状态的情况下启动）通常需要以下外部驱动数据：

数据类型 作用／为什么需要 常见格式 / 要素
全局模式“初始场 / 边界条件”（Initial Conditions & Lateral Boundary Conditions, IC/LBC） 用于给区域模式 (RAP) 的初始大气场、温度、湿度、风、地表状态等，以及边界的时间演变，用来驱动模型边界。 通常来自 GFS（或 NAM、GDAS 等）的大尺度气象场。格式 GRIB2 或 NEMSIO。包括多个气压层 + 地面变量 + 土壤温湿度等。
静态地表资料（Static / “fix” 文件） 地形、高程，土地利用 /覆盖类型，土壤类型，常年地面覆被（植被、水体、城市等），海岸线，常年气候背景（地表粗糙度、河网等） NetCDF、GRIB 或定制静态文件。
海表温度 / 冰盖 /海冰覆盖 /水面状态 用于边界面热量 /水汽通量 /反射率等地表–海面接口条件。 SST /冰覆盖 /海冰边界静态或近实况。
土壤状况 /土壤湿度 /土壤温度 /冻土状态 地表与地下热量和水分交换很关键，需要初始土壤温湿度 /温度分布。 多层土壤变量，有些资料可能在已有循环中，但冷启动时要从外部获取。
气象观测资料（用于分析／同化） 包括：地面观测（气温、湿度、压力、风速/风向）、高空气球 (radiosondes)、商用飞机报文（Aircraft reports：AMDAR 等）、雷达资料、卫星资料（包括卫星辐射、遥感水汽、云 /冰 /辐射相关观测） 多种格式（BUFR, GRIB, 特定卫星的 Level-1／2 辐射数据等）
遥感产品 / 雾 /云 /水汽 /辐射资料 用于改善云、水汽、水露点等变量的初始状态和同化。 卫星 /微波探测 /红外／可见光等。
气象分析背景 /气候气象背景统计项（误差协方差、背景场） 同化系统通常需要背景误差协方差、先验背景场、观测误差统计等，这些也算“驱动”或辅助资料。 通常来自历史资料或已有同化系统内部积累。

RAP 特定驱动 &关键来源

从文献和 NOAA/NCEP 的资料里可以看到 RAP 在冷启动或一般运行中常用以下关键来源：

  • RAP 的边界条件来源是 Global Forecast System (GFS) 模型。 NOAA Institutional Repository+2nco.ncep.noaa.gov+2
  • RAP 的同化（analysis）系统使用多种观测：surface stations、radiosondes（高空气球）、商用飞机、雷达（radar reflectivity & radial velocity 等）、卫星观察资料（水汽、云、辐射等） 美国气象学会期刊+2nco.ncep.noaa.gov+2
  • 静态地形 /地表覆盖等 fix 文件由 NOAA 提供。虽然我没找到公开 “RAP fix 文件完全包” 的清单细节，但在 UFS / SRW 这样的系统里，“fix” 数据集是必须预先下载的。其内容包括地形、土地覆盖、水体、海岸线、植被/植被类型、常年气候场等。 ufs-srweather-app.readthedocs.io+1

下载／获取方式

下面是这些数据通常可以从 NOAA 或关联平台获取的方法／链接：

数据／资料 来源／平台 获取方式（FTP / HTTP / API） 注意事项
GFS 模型数据（用于 IC / LBC） NOAA NOMADS / NWS / NCEP 公共模式数据仓库存档 HTTP 或 FTP，例如 NOMADS GRIB2 接口，也可能通过 AWS Open Data / S3 要确保选择正确的周期（cycle time），变量（温度、风、湿度等），所需层数，时间范围。冷启动通常需要过去某些时间的场（analysis）＋边界未来演变。
RAP 自身的分析／预报检验数据 NOAA NCEI（Rapid Refresh / RAP archive） HTTP / HTTPS / THREDDS 服务 /下载站点 若冷启动想对照或用做初始化，也可能用 RAP 的分析场。可能缺少某些观测驱动变量。 国家环境信息中心+1
观测资料（surface, radiosonde, aircraft, radar, satellite） 多个来源：NCEP / NWS / NOAA / NASA /卫星遥感机构 /地面观测网 /航空报文网络 通常通过 NOAA 的观测数据中心（如 NCEI, NWS）提供 BUFR 或其他格式；卫星资料通过 NOAA/NASA 的卫星数据仓库；雷达资料可能通过国防或气象雷达站网络 /NEXRAD /NOAA Radar Ops 等。 要注意时间戳对齐、质量控制、空间缺口等问题。对于冷启动，初始分析质量很敏感观测资料是否包含完整。
静态地表资料（terrain, land cover, soil type, etc.） NOAA / USGS /其他地面数据机构 下载“fix”文件集；可能 NOAA 提供；USGS 地形、土地覆盖数据库等 分辨率、格式要与模型网格匹配；可能需要投影/重网格处理。
海表温度 / 海冰覆盖 NOAA OISST /冰冻海洋数据 /卫星检验资料 NOAA 的 OISST 数据产品；冰情图 /冰盖产品；静态或近实况产品通过 NOAA /NASA 网站 要检查最新性、时间延迟；如果模拟包括海冰，条件要做得准确。
土壤温度／湿度等地表状态 如果已有循环则可从之前模拟中获取；否则可能需要外部气象再分析／重分析产品提供土壤状态（例如再分析资料） NOAA / NASA 再分析数据集（例如 NCEP Reanalysis, ERA5 等）或从类似 RAP archive 如果包含此类变量 冷启动时土壤状态可能是瓶颈，因为土壤调整／湿度分布可能很不准确，会导致地表–大气相互作用误差较大。

一个可能的冷启动流程（带下载步骤）

为了更清晰，这里举一个示例流程（假设你要冷启动 RAPv5，13-km 分辨率）：

  1. 从 NOAA / NOMADS 下载 GFS 的最新分析场（例如 00Z 分析）所需的变量：温度、风速（u, v）、湿度、水汽、气压、高度、地表压力、地面温度、土壤温度／湿度等。
  2. 下载静态地形 /地表资料 fix 文件集：地形边界，高程，土地覆盖类型，水体掩膜（land/water mask），植被类型等。
  3. 如果你的区域包含海岸或海面，获取最近的 SST /海冰掩膜资料。
  4. 获取观测资料以用于分析同化：地面气象站、飞机报文、高空气球、卫星水汽/云/辐射 /雷达反射率和径向速度等。
  5. 准备背景误差协方差 /先验背景场，如果你的同化系统需要这类静态或半静态资料。
  6. 将这些资料格式化为模型所需格式（GRIB2/NEMSIO/或者模型专用输入格式），包括插值到模型网格。
  7. 配置 namelist／模式启动控制文件（指定初始时间、边界条件频率、同化窗口等）。

初始场和边界条件

  • GFS 分析/预报场
    • 用作 RAP 冷启动的初始条件和外部边界条件。
    • 数据源：
      • NOAA NOMADS: https://nomads.ncep.noaa.gov
      • NCEI 数据归档: https://www.ncei.noaa.gov/
    • 文件格式：GRIB2

观测资料（用于分析/同化）

  • 地面观测（SYNOP、METAR、SHIP、BUOY）
  • 探空（RAOB）
  • 飞机观测（ACARS、AMDAR）
  • 卫星反演产品（GOES radiance、AMSU、IASI、ATMS 等）
  • 雷达资料（NEXRAD Level II/III，radial winds，reflectivity）
  • GPS 掩星、GPS PW
  • 航空模式专用：PIREPs
    • 获取方式：
      • NCEP Data Tank / NCEI: https://www.ncei.noaa.gov
      • MADIS（Meteorological Assimilation Data Ingest System）: https://madis.ncep.noaa.gov
      • NOAA AWS S3 buckets（含部分雷达/卫星数据）。
      • 归档观测可通过 AWS CLI 或 wget/curl 获取。

陆面/下垫面资料

  • 土地利用/地表类型（MODIS/USGS NLCD）
  • 土壤湿度/土壤温度（GFS 或 NLDAS）
  • 积雪数据（IMS Daily Snow, SNODAS）
  • 海表温度（RTG_SST, GHRSST）
    • 数据源：
      • NCEP RTG_SST (https://nomads.ncep.noaa.gov/pub/data/nccf/com/rtgsst/prod/)
      • NSIDC / NOAA IMS (雪盖数据)
      • NASA LAADS DAAC (MODIS 数据)

化学/气溶胶数据（若启用）

  • SMOKE 模型产物或 HRRR-Smoke 初值（火点、排放数据）
    • 来源：
      • NOAA ARL / EPA。
      • Fire INventory from NCAR (FINN)。
