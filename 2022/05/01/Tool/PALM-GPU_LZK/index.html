A100上编译运行plam模式

编译

  • palm_model_system-v23.10/packages/dynamic_driver/inifor/share/cmake/CMakeLists.txt中添加
  • palm_model_system-v23.10/packages/palm/model/share/cmake/CMakeLists.txt中添加 和
  • 因为palm依赖rrtmg，而rrtmg以共享库方式编译链接，nvfortran在编译openacc程序时，似乎不支持共享库用同样的编译选项，因此只能分开编译rrtmg和palm。可以把2中的CMakeLists.txt复制为2份，一份不包含-acc选项，然后在palm_model_system-v24.04/packages/palm/model/install中palmconfigure执行前分别 和
  • 为了增加constant_diffusion参数，在packages/palm/model/src/parin.f90中添加了一行
  • 23.10因为数据管理和代码有误（包括transpose_mod.f90和poisfft_mod.f90），不能支持cufft，因此不能打开__cuda_fft, 24.04可以打开该宏。
  • 24.04代码中明确写有routine not prepared for using openacc，因此在palm_model_system-v24.04/packages/palm/model/src/exchange_horiz_mod.f90中注释了类似的3行： 即使这样，运行还是报错，见下文。
  • 编译命令

测试

palmtest --cases openacc_test --builds nvidia_mpi_openacc_cufft --cores 1
其中用到了
palmrun -c "nvidia_mpi_openacc_cufft" -r openacc_test__nvidia_mpi_openacc_cufft__1 -a "d3#" -X 1 -B -v -z

运行（目前测试的情况）

  • 23.10（nvhpc）
    • 仅支持
    • 如果注释constant_diffusion = .T.，报错
  • 24.04（intel编译器cpu版本）
    • 只需要添加
  • 24.04（nvhpc）
    不能运行，代码本身有问题（没有做好数据管理），无论怎么设置，均提示：
